Overview

AJAXifies form including file uploads and stuff. Built on top of jQuery.Form.

Joosy.Form automatically cares of form validation highlights. It can read common server error responses and add .field_with_errors class to proper field.

If you don't have resource associated with form (#fill), it will try to find fields by exact keywords from response. Otherwise it will search for resource_name[field].

Examples:

Joosy.Form usage 

form = new Joosy.Form, -> (response)
  console.log "Saved and got some: #{response}"

form.progress = (percent) -> console.log "Uploaded by #{percent}%"
form.fill @resource

Constructor Details

# (void) constructor(form, options = {})

Note: If method of form differs from POST or GET it will simulate it
Note: For browsers having no support of HTML5 Forms it may do an iframe requests

During initialization replaces your basic form submit with AJAX request

by adding hidden _method input. In this cases the method itself will be set to POST.

to handle file uploading.

Parameters:

  • form (jQuery) — Form element instance
  • options (Hash) — Options

Options Hash: (options):

  • before (Function) — (XHR) -> Bollean to trigger right before submit. By default will run form invalidation cleanup. This behavior can be canceled by returning false from your own before callback. Both of callbacks will run if you return true.
  • success (Function) — (Object) -> null triggers on 2xx HTTP code from server. Pases in the parsed JSON.
  • progress (Function) — (Float) -> null runs periodically while form is uploading
  • error (Function) — (Object) -> Boolean triggers if server responded with anything but 2xx. By default will run form invalidation routine. This behavior can be canceled by returning false from your own error callback. Both of callbacks will run if you return true.
  • resource (Joosy.Resources.Base) — The resource to fill the form with
  • resourceName (String) — The string to use as a resource name prefix for fields to match invalidation
  • action (String) — Action URL for the form
  • method (String) — HTTP method, for example PUT, that will passed in _method param
  • debounce (Boolean) — Drop submit events while there is a pending submit request

# (Hash<String, Array>) __stringifyErrors(errors)

Prepares server response for default error handler Turns all possible response notations into form notation (foo[bar]) Every direct field of incoming data will be decorated by @substitutions

Examples:

Flat validation result 

{ field1: ['error'] }             # input
{ field1: ['error'] }             # if form was not associated with Resource by {#fill}
{ "fluffy[field1]": ['error']}    # if form was associated with Resource (named fluffy)

Complex validation result 

{ foo: { bar: { baz: ['error'] } } }    # input
{ "foo[bar][bar]": ['error'] }          # output

Parameters:

  • errors (Object) — Data to prepare

Returns:

  • (Hash<String, Array>) — Flat hash with field names in keys and arrays of errors in values
